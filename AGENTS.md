# Project Rules

This repository is a Quarto website for personal notes.

## Source Files

- Use English names for all folders and files.
- Use kebab-case for note filenames, for example `causal-inference.qmd`.
- Write source notes as `.qmd` files under `src/`.
- Do not edit files in `_site/`, `.quarto/`, or `*_files/`; they are generated by Quarto.
- Chinese or bilingual note titles are allowed in YAML front matter.
- Use `draft: true` in a note's YAML header to keep it unpublished.

## Quarto Notes

- Prefer Quarto Markdown and standard Markdown links over Obsidian-only syntax.
- Convert Obsidian wikilinks like `[[my-note]]` to regular links like `[My Note](my-note.qmd)`.
- Use relative links between notes.
- Keep images under `assets/images/` and other attachments under `assets/attachments/`.
- Keep bibliography files under `resources/bibliography/`.

## Migration Checklist

1. Rename the source file to an English kebab-case `.qmd` filename.
2. Add YAML front matter with at least a `title`.
3. Add `draft: true` while the note is still private or unfinished.
4. Convert Obsidian wikilinks, embeds, callouts, and attachment paths to Quarto-compatible Markdown.
5. Move images and attachments into `assets/`.
6. Run `quarto render` and fix warnings before committing.

## Git

- Commit source files and project documentation.
- Do not commit rendered output or local caches.
- Check `git status --short` before committing.
